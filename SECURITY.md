# Security policy

Do not open a public issue containing traces, prompts, responses, code, secrets,
credentials, object locators, presigned URLs, diagnostics archives, or raw logs.

Report suspected vulnerabilities privately through GitHub's private security
advisory flow for `smashah/traicer`. Include the affected commit, platform,
minimal reproduction steps, and sanitised impact. Use synthetic canaries rather
than real provider or customer data.

We aim to acknowledge a complete report within three working days, provide an
initial severity assessment within seven, and coordinate remediation and
disclosure according to exploitability and seller impact. These are response
targets, not a bug-bounty or payment commitment.
